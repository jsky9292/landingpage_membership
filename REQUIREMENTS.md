# 프로젝트 핵심 요구사항

## 콘텐츠 작성 원칙

### 1. AI스럽지 않은 글
- 기계적인 문장 패턴 사용 금지
- 자연스러운 구어체 표현 사용
- 템플릿화된 문구 지양

### 2. 경험에서 나오는 글
- 실제 사례 기반 작성
- 구체적인 숫자와 결과 명시
- 추상적 표현보다 실감나는 묘사

### 3. 후킹성 글
- 첫 문장에서 관심 유발
- 독자의 고민/문제점 직접 언급
- 결과 먼저, 과정은 나중에

### 4. 이모지 사용 금지
- 모든 콘텐츠에서 이모지 제외
- 텍스트만으로 임팩트 전달
- 깔끔하고 전문적인 톤 유지

## 시스템 요구사항

### 5. 로컬/온라인 데이터 동기화
- 로컬 개발환경과 배포환경 데이터 일치
- 환경별 설정 분리 및 관리
- 데이터베이스 마이그레이션 자동화

### 6. 관리자 화면 데이터 일관성
- 다른 PC에서 접속해도 동일한 데이터 표시
- 실시간 데이터 동기화
- 캐시로 인한 불일치 방지
- 세션 기반 일관된 사용자 경험
